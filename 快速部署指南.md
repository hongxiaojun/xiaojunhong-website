# 快速部署指南 - xiaojunhong.space

## 📋 部署前的准备清单

在开始之前，确保你有：
- ✅ GitHub 账户（免费创建）
- ✅ Cloudflare 账户（免费创建）
- ✅ Namecheap 账户（已有域名 xiaojunhong.space）
- ✅ 已安装 Hugo（运行 `brew install hugo`）

## 第一步：测试网站（本地）

```bash
# 进入项目目录
cd /Users/add/xiaojunhong-website

# 启动本地服务器
./setup.sh

# 或者手动启动
hugo server -D
```

在浏览器打开：http://localhost:1313

看到网站后，按 `Ctrl+C` 停止服务器。

## 第二步：创建 GitHub 仓库

### 2.1 在 GitHub 创建仓库

1. 访问 https://github.com/new
2. 仓库名称：`xiaojunhong-website`
3. 选择 **Public**（公开，Cloudflare Pages 免费版要求）
4. **不要**勾选 "Add a README file"
5. 点击 "Create repository"

### 2.2 推送代码到 GitHub

```bash
# 运行 GitHub 设置脚本
chmod +x github-setup.sh
./github-setup.sh

# 或者手动执行：
git branch -M main
git remote add origin https://github.com/你的用户名/xiaojunhong-website.git
git push -u origin main
```

## 第三步：连接 Cloudflare Pages

### 3.1 登录 Cloudflare

1. 访问 https://dash.cloudflare.com/
2. 如果没有账户，先注册（免费）
3. 登录后，在左侧菜单找到 "Workers & Pages"

### 3.2 创建 Pages 项目

1. 点击 "Create application"
2. 选择 "Pages" 标签
3. 点击 "Connect to Git"
4. 授权 Cloudflare 访问你的 GitHub
5. 选择 `xiaojunhong-website` 仓库
6. 点击 "Begin setup"

### 3.3 配置构建设置

在设置页面填写：

| 设置项 | 值 |
|--------|-----|
| **项目名称** | xiaojunhong-website |
| **生产分支** | main |
| **构建命令** | hugo |
| **构建输出目录** | public |
| **根目录** | (留空) |

点击 "Save and Deploy"

等待几分钟，Cloudflare 会给你一个临时网址：
```
https://xiaojunhong-website.pages.dev
```

访问这个网址，你应该能看到你的网站！

## 第四步：绑定域名 xiaojunhong.space

### 4.1 在 Cloudflare Pages 添加自定义域名

1. 在你的 Pages 项目中，点击 "Custom domains"
2. 点击 "Set up a custom domain"
3. 输入：`xiaojunhong.space`
4. 点击 "Continue"

Cloudflare 会显示需要添加的 DNS 记录。

### 4.2 在 Namecheap 配置 DNS

1. 登录 https://ap.www.namecheap.com/
2. 进入 "Domain List" → 找到 "xiaojunhong.space" → 点击 "Manage"
3. 点击 "Advanced DNS" 标签
4. 删除现有的记录（如果有）
5. 点击 "Add New Record"，添加两条记录：

**记录 1：**
- Type: `CNAME`
- Name: `@`
- Value: `xiaojunhong-website.pages.dev`
- TTL: Automatic

**记录 2：**
- Type: `CNAME`
- Name: `www`
- Value: `xiaojunhong-website.pages.dev`
- TTL: Automatic

6. 点击 "Save All Changes"

### 4.3 等待 DNS 生效

DNS 传播通常需要 5-30 分钟，最多可能需要 24 小时。

检查 DNS 状态：https://www.whatsmydns.net/

生效后，访问 https://xiaojunhong.space 就能看到你的网站！

## 第五步：更新网站内容

### 创建新文章

```bash
# 使用快捷脚本
chmod +x new-article.sh
./new-article.sh article "我的新文章"

# 或者使用 Hugo 命令
hugo new articles/my-new-article.md
```

编辑创建的 Markdown 文件：
```bash
nano content/articles/my-new-article.md
```

### 发布更新

```bash
# 添加更改
git add .

# 提交更改
git commit -m "添加新文章"

# 推送到 GitHub
git push
```

Cloudflare 会自动检测推送并重新部署网站！

## 常见问题解决

### 问题 1：网站无法加载

**检查步骤：**
1. 确认 Cloudflare Pages 部署成功（查看部署日志）
2. 确认 GitHub 仓库是公开的
3. 确认 DNS 设置正确

### 问题 2：构建失败

**解决方法：**
1. 在 Cloudflare Pages 查看构建日志
2. 本地测试：`hugo` 命令是否正常
3. 检查 config.toml 语法是否正确

### 问题 3：域名无法访问

**解决方法：**
1. DNS 传播需要时间，等待 30 分钟
2. 在 Namecheap 确认 DNS 记录正确
3. 使用 DNS 检查工具验证：https://www.whatsmydns.net/

### 问题 4：图片不显示

**解决方法：**
1. 将图片放在 `static/images/` 目录
2. 在 Markdown 中引用：`![描述](/images/文件名.jpg)`
3. 注意路径前面有 `/`
